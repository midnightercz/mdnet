# Search Feature

MDNet includes a powerful search system that indexes all your markdown pages.

## How to Search

Click the **🔍** button in the top-right corner to open the search modal.

### Text Search

Simply type any text to search for pages by title or heading:

- Type `"example"` to find pages with "example" in the title or headings
- Type `"feature"` to find pages about features
- Type `"layout"` to find pages about layouts

### Tag Search

Use the `#` prefix to search for tags:

- Type `#tag1` to find all pages tagged with "tag1"
- Type `#search` to find pages about search (including this page!)
- Type `#example` to find example pages

### Property Search

Search by front matter properties using `property:operator:value` syntax:

**Operators:**
- `==` (or empty) - equals
- `!=` - not equals
- `has` - contains (for arrays/lists)
- `!has` - does not contain

**Examples:**
- `title:Search Feature` - pages with title "Search Feature"
- `author:==John` - pages where author equals "John"
- `status:!=draft` - pages where status is not "draft"
- `md-tags:has:search` - pages with "search" in tags
- `category:Tutorial` - pages in Tutorial category (defaults to ==)

## What Gets Indexed

The indexing system automatically extracts:

1. **Title** - From front matter or first H1 heading
2. **Headings** - All H1-H6 headings in the document
3. **Tags** - Both `#tag` mentions in text and `md-tags` from front matter
4. **Links** - All wiki-links `[[page]]` between pages
5. **Properties** - All front matter properties for advanced searching

## Search Results

Results show:
- **Page title** in bold
- **Matched heading or tag** in smaller text below

Results are paginated (10 per page) for easy navigation.

## Keyboard Shortcuts

- Press **Escape** to close the search modal
- Click outside the search box to close it

## Automatic Indexing

The backend server automatically manages the search index:

### On Server Startup
- Runs indexing immediately when the server starts
- Ensures the index is always up-to-date

### Periodic Re-indexing
- Server re-indexes content every **5 minutes** by default
- Only re-indexes files that have been modified
- Tracks modification times to avoid unnecessary work
- Removes deleted files from the index

### Configuration

Control the re-indexing interval with the `INDEX_INTERVAL` environment variable:

```bash
# Re-index every 10 minutes
INDEX_INTERVAL=600000 npm start

# Disable automatic re-indexing
INDEX_INTERVAL=0 npm start
```

### Manual Indexing

You can also manually rebuild the index:

```bash
npm run build:index
```

## Index Location

The search index is stored at:
- **Source**: Generated by `build-index.js`
- **Output**: `public/search-index.json`
- **State**: `.index-state.json` (tracks last indexed times)

Each indexed page includes:
- Filename, title, headings, tags, links
- **Timestamp** of when it was last indexed (`indexedAt` field)

## Try It Now!

1. Click the 🔍 button
2. Try searching for:
   - `"wiki"` - finds pages mentioning wiki
   - `#tag1` - finds pages tagged with tag1
   - `title:Search` - finds pages with "Search" in title
